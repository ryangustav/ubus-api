# Codebase Structure

**Analysis Date:** 2026-03-30

## Directory Layout

```
ubus-api/
├── .planning/          # GSD planning and codebase mapping
├── .github/            # CI/CD workflows (GitHub Actions)
├── drizzle/            # Database migrations and metadata
├── nginx/              # Reverse proxy configuration and certs
├── scripts/            # Database seeding and bootstrap scripts
├── src/                # Core application source code
│   ├── config/         # System-wide configuration (OCI, Vault)
│   ├── modules/        # Feature-based business modules
│   └── shared/         # Common utilities, guards, and services
├── test/               # E2E and unit test suites
├── Dockerfile          # Production container definition
├── docker-compose.yml  # Development orchestration
└── package.json        # Project manifest and dependencies
```

## Directory Purposes

**src/modules/**:
- Purpose: Feature-encapsulated business logic.
- Contains: `auth/`, `trips/`, `users/`, `fleet/`, `reservations/`, `management/`, `metrics/`.
- Structure: Each module typically follows a NestJS structure for Controllers, Services, and Modules.

**src/shared/**:
- Purpose: Reusable logic across multiple modules.
- Contains:
  - `database/`: Drizzle schemas and module.
  - `redis/`: Redis connection and module.
  - `queue/`: BullMQ configuration.
  - `guards/`: Global/shared auth and role guards.
  - `email/`: Transactional email service.

**drizzle/**:
- Purpose: Database versioning and schema history.
- Contains: Generated SQL migrations and `meta/` snapshots.

**nginx/**:
- Purpose: Infrastructure configuration for the reverse proxy.
- Contains: `conf.d/` for server blocks and `certs/` for SSL (if any).

**scripts/**:
- Purpose: Administrative and setup tasks.
- Contains: `seed-super-admins.ts`, `run-bootstrap.ts`.

## Key File Locations

**Entry Points:**
- `src/main.ts`: Main application bootstrap.
- `src/app.module.ts`: Root module orchestrating all features.

**Configuration:**
- `package.json`: Dependencies and script definitions.
- `drizzle.config.js`: Drizzle ORM configuration.
- `tsconfig.json`: TypeScript compiler options.
- `docker-compose.prod.yml`: Production container setup.

**Core Logic:**
- `src/modules/*/application/*.service.ts`: Business logic implementations.
- `src/shared/database/schema/*.schema.ts`: Database entity definitions.

**Testing:**
- `test/jest-e2e.json`: E2E test configuration.
- `test/app.e2e-spec.ts`: Application-wide E2E tests.

## Naming Conventions

**Files:**
- `kebab-case.ts`: General source files.
- `*.module.ts`, `*.service.ts`, `*.controller.ts`: NestJS architectural components.
- `*.schema.ts`: Drizzle table definitions.
- `*.sql`: Database migrations.

**Directories:**
- `kebab-case`: All directory names.
- `plural`: For collections like `modules/`, `shared/`, `scripts/`.

## Where to Add New Code

**New Feature/Module:**
- Definition: `src/modules/{feature}/{feature}.module.ts`.
- Implementation: `src/modules/{feature}/application/{feature}.service.ts`.
- API: `src/modules/{feature}/{feature}.controller.ts`.

**New Database Table:**
- Definition: `src/shared/database/schema/{table}.schema.ts`.
- Export: Add to `src/shared/database/schema/index.ts`.
- Migration: Run `npm run db:generate`.

**New Shared Utility:**
- Implementation: `src/shared/{utility-name}/`.

## Special Directories

**.planning/**:
- Purpose: Contains GSD workflow state and codebase mapping.
- Source: Generated by GSD agents/workflow.
- Committed: Yes (standard) or No (if `commit_docs: false`).
